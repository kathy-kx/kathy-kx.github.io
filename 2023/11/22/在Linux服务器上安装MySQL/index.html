在Linux服务器上安装MySQL

发表于 2023-11-22 更新于 2023-11-23 分类于 notes

首先检测当前系统中是否安装MySQL数据库。
rpm -qa 查询当前系统中安装的所有软件。qa意思是query all
rpm -qa | grep mysql 查询当前系统中安装的名称带mysql的软件
rpm -qa | grep mariadb 查询当前系统中安装的名称带mariadb的软件

RPM (Red-Hat Package Manager)RPM软件包管理器，是红帽Linux用于管理和安装软件的工具。
注：如果安装过了mariadb，需要先卸载。否则mysql会安装失败。
卸载软件命令：rpm -e --nodeps 软件名称。例如rpm -e --nodeps mariadb-libs-xxxxxxx

安装和配置MySQL的步骤

  1. 下载mysql8.0.31 arm64版本（适用于macOS m2芯片）。mysql-8.0.31-linux-glibc2.17-aarch64.tar.gz
    下载地址：https://downloads.mysql.com/archives/community/

  2. 将下载好的mySQL安装包上传到linux并解压

  • 在本地终端，用sftp传输该安装包
    put /Users/kathy/Downloads/linux-software/mysql-8.0.31-linux-glibc2.17-aarch64.tar.gz /root/

  • 在虚拟机终端，解压安装包到/usr/local/mysql(先用mkdir mysql创建好)
    tar -zxvf mysql-8.0.31-linux-glibc2.17-aarch64.tar.gz -C /usr/local/mysql

  • 在虚拟机终端，在/usr/local/mysql-8.0.31-linux-glibc2.17-aarch64/下，创建包data和log

  1. 修改环境变量：
    cd ~

vim .bash_profile

其中添加：PATH=$PATH:$HOME/bin:/usr/local/mysql-8.0.31-linux-glibc2.17-aarch64/bin

修改完后source .bash_profile

which mysql - 会显示/usr/local/mysql-8.0.31-linux-glibc2.17-aarch64/bin/mysql

  1. 创建mysql用户和用户组 groupadd mysql

useradd -r -g mysql -s /bin/false mysql

chown -R mysql:mysql /usr/local/mysql-8.0.31-linux-glibc2.17-aarch64/

sudo mkdir /var/lib/mysql

sudo chown -R mysql:mysql /var/lib/mysql

  1. 配置参数文件

vim /etc/my.cnf

  1. 启动MySQL的mysqld服务，并使用指定的参数进行初始化

执行命令：/usr/local/mysql-8.0.31-linux-glibc2.17-aarch64/bin/mysqld --initialize --user=mysql --basedir=/usr/local/mysql-8.0.31-linux-glibc2.17-aarch64 --datadir=/usr/local/mysql-8.0.31-linux-glibc2.17-aarch64/data

具体含义：

  • /usr/local/mysql-8.0.31-linux-glibc2.17-aarch64/bin/mysqld-启动MySQL的mysqld服务
  • -initialize - 初始化MySQL数据目录
  • -user=mysql - 使用mysql用户运行mysqld服务
  • -basedir=/usr/local/mysql-8.0.31-linux-glibc2.17-aarch64 - 指定MySQL的安装根目录
  • -datadir=/usr/local/mysql-8.0.31-linux-glibc2.17-aarch64/data - 指定MySQL数据目录

还会生成随机密码，可以进入log日志目录查看密码：
cd /usr/local/mysql-8.0.31-linux-glibc2.17-aarch64/log log是刚才创建的日志目录
cat error.err

可以看到临时root密码，记录下来：
A temporary password is generated for root@localhost: **********

  1. 添加mysqld到systemd管理
    vim /usr/lib/systemd/system/mysqld.service
  1. 将mysqld将入systemd管理，开机自启动mysqld

systemctl daemon-reload 重新加载systemd管理的所有配置文件，使之生效

systemctl start mysqld 启动MySQL服务（服务名就叫mysqld）。

systemctl enable mysqld 设置为开机自启动。

执行后，显示Created symlink /etc/systemd/system/multi-user.target.wants/mysqld.service → /usr/lib/systemd/system/mysqld.service.

  1. 查看 MySQL 服务状态
    systemctl status mysqld 查看 MySQL 服务状态的命令（正常运行），可以查看 MySQL 服务是否正在运行，服务的进程 ID，以及最近一次服务状态的变化时间等信息。如果 MySQL 服务未能正常启动，该命令也可以提供错误信息，帮助诊断问题。

查看已经启动的服务（非必需）netstat -tunlp

  1. 登陆 MySQL 数据库：

mysql -u root -p后，此时我这里出现错误：
mysql: error while loading shared libraries: libncurses.so.5: cannot open shared object file: No such file or directory
意思是mysql需要libtinfo.so.5这个库文件，但是找不到。

解决方法：ln -s /usr/lib64/libtinfo.so.6 /usr/lib64/libtinfo.so.5

  1. 修改密码

切换到 MySQL 数据库，并使用以下命令修改 root 用户的密码：（注意要有分号，符号全是小写）

use mysql;

  • 开启访问权限（非必需）

CREATE USER ‘root‘@’%’ IDENTIFIED BY ‘PASSWORD’; （%是匹配所有的ip地址）

GRANT ALL PRIVILEGES ON *.* TO ‘root‘@’%’ WITH GRANT OPTION;

flush privileges;

  • 修改密码

alter user root@”localhost” identified with mysql_native_password by ‘新密码’;

修改密码后，使用以下命令刷新 MySQL 权限：flush privileges;

  1. 退出 MySQL 数据库
    mysql> exit，并使用以下命令重启 MySQL 服务：systemctl restart mysqld

  2. 从外部连接：
    打开navicat，新建连接（Connection - mySQL），连接名任意，主机或IP地址填虚拟机端的IP地址192.168.50.158，用户名root，密码*****。

将虚拟机端端口3306开放，firewall-cmd --zone=public --add-port=3306/tcp --permanent以及firewall-cmd --reload。

双击新建的连接，即可连接上。
