git常用命令

发表于 2023-11-21 更新于 2023-11-23 分类于 notes

git全局设置

安装git后（macOS安装Xcode即可），就可以设置用户信息(用户名和邮箱)。每次git提交时都会使用该用户信息。

设置用户信息

打开终端，输入

git config –global user.name “随便写用户名 只是一个标识符”

例如，~ % git config --global user.name "kxzhu"

git config –global user.email “随便写邮箱”

例如，~ % git config --global user.email "xxxxxx@qq.com"

查看配置信息

git config --list

获取git仓库

要使用git进行版本控制，首先要获取git仓库

获取git仓库有两种方式：

  • 在本地初始化一个git仓库（不常用）

    • 在本地任意目录下创建一个空目录，作为本地git仓库
    • 从终端cd到该目录，执行命令git init

    成功后，该空目录中会生成隐藏文件.git，mac使用“command + shift + 句号”，可显示隐藏文件

  • 从远程仓库克隆（常用）

    • 在终端cd到一个新的目录，不要在刚才本地git仓库目录下操作。git仓库不能嵌套！

    • 在gitee刚才创建的仓库页面，点HTTPS，复制链接

    • 执行命令git clone [复制的远程仓库地址]

      例如git clone https://gitee.com/kxzhu93/hellogit.git

    • 输入gitee的账号密码，完成

    成功后，在刚才的目录下，会克隆一个远程仓库对应的文件夹，里面也有.git隐藏文件

工作区、暂存区和仓库

版本库：.git隐藏文件夹

工作区：包含.git文件夹的目录

暂存区：在工作区(hellogit仓库对应的文件夹hellogit)中新建一个test.java文件，在终端cd到hellogit文件夹，再执行git add *命令，将文件的修改放在暂存区。则.git文件夹中会出现index文件，这就是暂存区。

后续可以执行git commit命令，将文件的修改提交到版本库

工作区、暂存区和仓库之间的状态转换：

  • git add (工作区 –> 暂存区)
  • git commit (暂存区 –> 本地仓库)

文件状态

Git工作目录下对于文件的修改(增加、删除、更新)会存在几个状态，这些修改的状态会随着我们执行Git 的命令而发生变化。

查看文件状态：git status

过程：

  • 在工作区新创建一个文件，该文件状态为untracked；

    在工作区修改已有的文件，该文件状态为unstaged；

  • 执行git add后，将工作区的文件加入暂存区，属于已跟踪tracked的文件，状态为

    • unmodified 已跟踪的文件没修改，提交到了仓库，是unmodified
    • modified 已跟踪的文件被修改了，但又没有add到暂存区时，是红色modified。
      已跟踪的文件被修改了，又add到暂存区时，绿色modified。
    • staged 已暂存

  • 执行git commit后，将暂存区的文件提交到仓库

本地仓库常用命令

git status查看文件状态

git add将文件的修改加入暂存区

git reset将暂存区的文件取消暂存，或者是切换到指定版本

取消暂存：

例如`git reset test.xml`之后，刚才加入暂存区的test.xml又回到Untracked状态

切换到指定版本：

例如`git reset --hard 版本号xxxxxxxxx`，可以去想去的版本

git commit将暂存区的文件修改提交到版本库

例如`git commit -m "注释内容" test.java`

git log查看日志

能看到谁提交的、版本号

远程仓库常用命令

  1. git remote 查看远程仓库

    如果想查看已经配置的远程仓库服务器，可以运行 git remote 命令，它会列出每一个远程服务器的简写。如果已经克隆了远程仓库，那么至少应该能看到 origin，这是 Git 克隆的仓库服务器的默认名字。

    精确的可以用git remote -v

  2. git remote add 添加远程仓库

    将本地仓库(本地目录使用git init创建的仓库)和远程仓库(gitee上创建的同名仓库)关联上。后续本地和远程就可以通信（可以拉取和推送）

    语法是：git remote add <shortname> url

    例如git remote add origin https://gitee.com/kxzhu93/git_test.git

  3. git clone 从远程仓库克隆

    功能：把远程仓库复制一份

    语法是：git clone <url>

    例子：

    • 在gitee上点了“初始化仓库”后，会创建分支（当前是master）和readme.md文件
    • 新建一个文件夹-cd到该文件夹-执行git clone 远程url

    （例如git clone https://gitee.com/kxzhu93/git_test.git）-该新建的文件夹内就会有克隆下来的仓库

  4. git pull 从远程仓库拉取

    从远程仓库获取最新版本，并合并到本地仓库，可以使用命令: git pull [short-name][branch-name]

    过程：

    • 别人修改了文件，推送到远程仓库
    • 你可以执行git pull origin master从远程仓库拉取，获取新的版本

    注：如果当前本地仓库不是从远程仓库克隆，而是(通过git init)本地创建的仓库，并且仓库中存在文件，此时再从远程仓库拉取文件的时候，会报错 (fatal: refusing to merge unrelated histories）即远程仓库和本地仓库都有各自的内容-无关的历史，
    解决此问题可以在git pull命令后加入参数–allow-unrelated-histories，git pull origin master --allow-unrelated-histories，就会打开一个编辑框，输入i - 随便编辑一些信息 - esc - :wq，就可以合并远程仓库和本地仓库，都拉取下来。如果远程没有、本地有的，可以push上去

  5. git push 推送到远程仓库

    将本地仓库内容推送到远程仓库，可以使用命令:git push[remote-name][branch-name]

    过程：

    • 在本地工作区新增/修改了test.txt文件
    • 执行git add test.txt添加到暂存区
    • 执行git commit -m "init test.txt" * 添加到本地仓库（引号内是注释）
    • 执行git push origin master推送到远程仓库

分支

简介

使用分支，意味着你可以把你的工作从开发主线上分离开来，以免影响开发主线。

同一个仓库可以有多个分支，各个分支相互独立，互不干扰。

通过git init 命令创建本地仓库时，默认会创建一个master分支。

分支的相关命令

  1. 查看分支
  • git branch 列出所有本地分支
  • git branch -r 列出所有远程分支(remote)
  • git branch -a 列出所有本地分支和远程分支(all)

在终端显示的时候，绿色、前面有*号的，是当前所在的分支。其他分支是白色。

  1. 创建分支

    git branch [分支名称]
    例如，git branch b1，可以创建名为b1的新分支
    此时创建的是本地分支

  2. 切换分支
    git checkout [分支名称]
    例如，git checkout b1，或者git checkout master

  3. 推送至远程仓库分支
    使用git branch [分支名称]创建的是本地分支，还需要推送到远程仓库，否则远程仓库不会显示新创建的分支。命令为：
    git push [shortName] [name]
    例如，git push origin b1，将创建的本地分支b1(及该分支下已经commit的文件)推送到远程仓库

  4. 合并分支
    git merge [name]
    把b1分支、b2分支下的内容都合并到master主分支：

    • 切换回主分支git checkout master
    • 合并b1 git merge b1，进入了编辑窗口，输入:wq保存退出（如需编辑，就输入i - 编辑 - esc - :wq），回车后b1就合并到master了
    • 合并b2 git merge b2，进入了编辑窗口，输入:wq保存退出，b2也合并到master了

合并分支时解决冲突

在两个分支中都修改了同一个文件，在合并时可能出现问题。

  • 先手动解决冲突，修改文件中出现提示信息的位置。
  • 再使用git add [文件名]，告诉git已经手动处理了该文件的冲突，并可以添加到暂存区
  • 提交到仓库git commit -m “注释” [文件名]

如果出现错误“fatal: cannot do a partial commit during a merge”，可以使用git commit -m “注释” [文件名] -i

  • push到远程仓库git push origin master

标签

常用来记录发布版本的节点。把当前分支的内容全部记录下来了。可以理解为某个状态的一张照片。

分支是可以推进、变化的，标签是记录下来的静态、固定的。

  1. 列出已有的标签 git tag

  2. 创建新的标签 git tag [name]

    例如git tag v0.1

  3. 将标签推送至远程仓库 git push [shortName] [name]

    例如git push origin v0.1

    新增文件、加标签并推送的完整过程是：

    • 在本地工作区手动修改/添加了某文件
    • git add *添加到暂存区
    • git commit -m "init test2.txt" test2.txt 提交到本地仓库
    • git tag v0.2为现在的状态创建新的标签
    • git push origin v0.2 将标签及当前状态推送到远程仓库

  4. 检出标签 git checkout -b [branch] [name]

    创建一个新的分支指向某个标签。

    例如，git checkout -b b0.1 v0.1，使用命令后，会自动切换到新的分支b0.1，且记录的是v0.1这个标签
