    IDEA中使用git

    Posted on 2023-11-21 Edited on 2023-11-30 In notes

    一、将IDEA上自己创建的项目上传到远程仓库：

    1、本地初始化仓库

    用IDEA创建maven项目，希望通过本地git仓库对该项目进行管理

    先创建好maven项目；

    顶部菜单VCS(Version Control) - Create Git Repository - 选择你的项目目录后，该项目目录就会变成仓库目录 - IDEA顶部也会出现几个git图标

    2、将本地仓库关联gitee上的远程仓库

    右键项目名 - Git - Manage Remotes - 在弹出的Git Remotes点击“+”，输入name(例如origin) & url(例如https://gitee.com/kxzhu93/git_test.git) 即可让本地仓库关联该远程仓库；

    3、将本地IDEA中修改的内容add、commit(选Unversioned Files)、push到远程仓库（使用git图标即可）

    二、从远程仓库克隆项目，在本地修改后，再推送到远程仓库

    1、远程克隆仓库

    克隆了gitee上的一个仓库，希望在本地查看、修改、推送

    方法：

    • IDEA顶部菜单VCS(Version Control) - Get from Version Control - 输入远程仓库复制的的url地址(https) - 输入gitee的账号密码 - 即可克隆
    • 或者从刚进IDEA还没打开任何项目的页面 - Get from Version Control 也可以找到此页面
    • 或者点Git - clone - 也可以找到此页面

    2、将本地IDEA中修改的内容add、commit(选Unversioned Files)、push到远程仓库（使用git图标即可）

    三、已经关联了远程仓库，远程仓库的代码更新了，需要从仓库拉取

    方法：点git工具图标的“update project” - ok

    配置

    安装git之后，在IDEA的设置-version control - Git - Path to git executable处会自动检测到git安装位置。点击右侧Test，如果出现了版本号，说明位置没问题。

    获取git仓库

    本地初始化仓库

    用IDEA创建maven项目，现在希望通过git仓库对该项目进行管理

    方法：顶部菜单VCS(Version Control) - Create Git Repository - 选择你的项目目录后，该项目目录就会变成仓库目录 - IDEA顶部也会出现几个git图标

    远程克隆仓库（常用）

    克隆了gitee上的一个仓库，希望在本地查看、修改、推送

    方法：

    • IDEA顶部菜单VCS(Version Control) - Get from Version Control - 输入远程仓库复制的的url地址(https) - 输入gitee的账号密码 - 即可克隆
    • 或者从刚进IDEA还没打开任何项目的页面 - Get from Version Control 也可以找到此页面
    • 或者点Git - clone - 也可以找到此页面

    .gitignore文件

    作用：定义当前仓库里哪些文件不需要git管理。例如.git, target文件夹, .iml文件等都不需要git管理

    如何创建：在gitee创建远程仓库时，选择“初始化仓库” ，语言选择Java，添加.gitignore选择Java，这样创建后仓库根目录就会有该文件

    本地仓库操作

    将文件加入暂存区

    方法1: 新创建一个类，弹出的对话框点“add“，就是将该类加到了暂存区。相当于git add命令。

    方法2: 如果没有点“add”，可以选中文件，点右键-Git-Add

    红色的是没加到暂存区

    绿色的是加到暂存区了

    黑色的是已经纳入git管理

    将暂存区的文件提交到版本库(提交到本地仓库)

    相当于git commit -m “注释”

    方法1：选中文件，点右键-Git-Commit File - 选中要提交的文件 - Commit Message中可以输入注释

    方法2: 上方git工具中点绿色对勾。相当于：加到暂存区+提交

    查看日志

    相当于git log

    方法：上方git工具中点时钟按钮（show history）可以看本地修改的(Local Changes)、整个项目所有修改的(Log)。可以点每个版本修改了哪里

    远程仓库操作

    查看当前项目关联的远程仓库

    右键项目名 - Git - Manage Remotes

    添加远程仓库

    将本地仓库关联gitee上的远程仓库。

    方法：右键项目名 - Git - Manage Remotes - 在弹出的Git Remotes点击“+”，输入name(例如origin) & url(例如https://gitee.com/kxzhu93/git_test.git) 即可让本地仓库关联该远程仓库

    一个本地仓库可以对应多个远程仓库

    推送至远程仓库

    方法：右键项目名 - Git - Push

    master→origin:master的意思是，将本地仓库的master分支推送到远程仓库origin的master分支

    从远程仓库拉取

    已经关联了远程仓库，远程仓库的代码更新了，想要从仓库拉取下来

    方法：git - pull

    或者点git工具图标的“update project”

    可以从Remote这一项，看到从哪个远程仓库、哪个分支拉取

    分支操作

    查看分支

    方法：git-braches

    或者IDEA右下角会显示分支

    创建分支

    IDEA右下角点分支 - new branch

    如果选择“checkout branch”，就是创建的同时切换到该新分支

    切换分支

    IDEA右下角点分支 - 在local branches中选择要切换的分支 - checkout切换

    将分支推送到远程仓库

    希望将本地仓库创建的分支推送到远程仓库

    方法：IDEA右下角点分支 - 选择要推送的分支 - 点push

    合并分支

    IDEA右下角先切换到你的主分支 - 点要合并的分支 - merge into current(例如merge ‘b1’ into ‘main’，将b1分支合并到main分支中)
